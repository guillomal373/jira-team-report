Customer Service Rule

All issues in Call Agent and Completed must be handled by Customer Service.

After they attend the case, they must change the status to Resolved with CX or Unresolved with CX.

Quick Tips

What To Collect Before Escalating

These inputs reduce loops between Triage, Call Agent, and In progress.

Identity and Context

  • Customer ID or account ID exactly as entered.
  • Platform and device: iOS or Android, plus phone model when relevant.
  • Flow stage: login, first login, blocked list, settings, carrier setup, or password reset.

Error Evidence

  • Exact error message, not a paraphrase.
  • Screenshot or short screen recording when the issue is visual or intermittent.
  • Date and approximate time when the user reproduced the problem.

Validation Checks

  • Confirm if the user already tried reinstalling, logging out, or resetting password.
  • Check whether phone number, DOB, and last 4 of SSN were validated against the expected source.
  • Note if the issue was reproduced internally or only reported by the customer.

Comment Quality

  • Write the next action in the first sentence.
  • Separate facts from assumptions.
  • Link the Jira ticket when engineering work already exists.
Instructions

How To Use The Statuses

Use statuses as operational signals, not just labels. Each one should imply a specific next move.

New

The issue was received but has not been evaluated yet.

Triage

Initial review is in progress and ownership or category is being decided.

Call Agent

More information is required from the reporting side before meaningful work can continue.

In queue

The issue is accepted and waiting for engineering or operational execution.

In progress

Someone is actively testing, fixing, validating, or coordinating the solution.

Follow up

Additional confirmation is pending after a fix, reply, or change was already provided.

Completed

The internal work is done and the expected action or recommendation was delivered.

Resolved with cx

The customer impact is confirmed as solved.

Unresolved with cx

The customer still reports the issue or the expected outcome was not achieved.

General Back to Issue Structure

Errors without information

Ask the customer to provide more information about the flow where the error occurs.

  • Ask what information was entered, if applicable.
  • Ask for phone details, including device model and platform.
  • Do not escalate until the missing context is captured.
First Login Back to Issue Structure

Associated phone number is missing

This happens when the user does not have a mobile phone number registered in ForthCRM.

  • Open the customer record in ForthCRM.
  • Update the mobile phone number field.
  • Ask the customer to retry first login after the update.
First Login Back to Issue Structure

First time steps

Use this when the customer does not know how to activate the CreditorX account with Forth data.

  • Walk the customer through the first-login activation flow.
  • Confirm the customer is using the correct Forth information.
  • Stay with the customer until the account activation path is clear.
First Login Back to Issue Structure

Account already created

The customer cannot complete first login because an account already exists.

  • Tell the customer the account has already been created.
  • If they do not remember the password, help them start password recovery.
  • Do not send them through first-login activation again.
First Login Back to Issue Structure

Invalid credentials - Account ID (External ID)

The related customer ID may not exist in the Forth database as entered.

  • Validate the correct account ID in Forth.
  • Tell the customer which account ID should be used.
  • Have the customer retry login with the validated account ID.
First Login Back to Issue Structure

Cannot create the password

Verify that the password matches the required security format.

  • Use more than 8 characters.
  • Include uppercase, lowercase, numbers, and special characters.
  • If it still fails, ask the customer to reinstall the app and retry.
Redirecting Setup Back to Issue Structure

Carriers and Providers

If the customer uses a different mobile carrier, including an MVNO such as Cricket Wireless or Spectrum, help them select the corresponding host network carrier.

  • Ask the customer for their provider name.
  • Search the carrier/provider mapping from the guide.
  • Tell the customer which host network carrier to select.
  • Continue into USSD setup if call forwarding still is not configured.
Redirecting Setup Back to Issue Structure

Calls are not being forwarded

Complete the forwarding checklist with the customer before escalating.

  • Confirm the app has permission to be the default caller ID app.
  • Go to Settings and enable the first two Inbound Call Settings checkboxes.
  • Confirm suggested numbers appear in Blocked and are blocked in the app.
  • Use More if needed to expose the verified numbers in Suggestions.
  • Select the carrier and execute the USSD commands.
  • Confirm the commands appear green and Continue is enabled.
Redirecting Setup Back to Issue Structure

Sending USSD commands

Use this when carrier codes were not configured when the customer entered the platform.

  • Open the carrier code setup flow with the customer.
  • Follow the step-by-step instructions for the required carrier codes.
  • Verify blocked numbers and call diversion after setup.
Redirecting Setup Back to Issue Structure

USSD codes will not allow checkbox click

The known test on iPhone 14 completed correctly, so collect device-specific information for accurate testing.

  • Ask which device model and OS version the customer is using.
  • Confirm exactly when the checkbox remains disabled.
  • Escalate with device details and the point where the flow blocks.
Login Back to Issue Structure

Does not know the username

The username may be the email address, account ID, or phone number, depending on registration.

  • Ask the customer to try all three possible username values.
  • If they do not remember the information, validate it in Forth.
  • Provide the correct credential value after validation.
Login Back to Issue Structure

Reset password

Use this when login fails because the customer forgot credentials.

  • Start the password reset process.
  • Use the correct username, normally email or account ID.
  • Make sure the requested date format is followed.
Login Back to Issue Structure

Frozen screen or sudden close

Start with a clean reinstall and repeat the initial login process.

  • Ask the customer to reinstall the application.
  • Have them repeat the initial login process.
  • If it continues, collect phone model and prior troubleshooting steps.
Logout Back to Issue Structure

Unexpected logout

An unexpected logout can happen when the same session is opened on another device.

  • Explain that logging in on a new device closes the previous device session.
  • Ask the customer to verify the session is not open elsewhere.
  • Ask whether anyone else may have access to the account.
